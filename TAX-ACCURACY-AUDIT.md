# Tax Accuracy Audit тАФ 2026-07-06

Researched via three parallel passes (UK/IE/DE, FR/NL/ES, US/CA/AU) against current official sources, checked line-by-line against the exact code in `index.html`'s `COUNTRIES` object. Findings below split into **structural bugs** (wrong regardless of tax year) and **stale figures** (right shape, old numbers). The app's own disclaimer ("simplified effective-rate estimates... ignores allowances, credits, student loans, overtime rules") is a reasonable simplification boundary тАФ nothing here asks for full statutory precision, only for the core brackets/rates to be structurally correct and current.

## Structural bugs (wrong at any tax year, not just outdated)

1. **France's bracket loop is shifted by one row.** The array `[[11294,0],[28797,.11],[82341,.30],[177106,.41]]` pairs each threshold with the *wrong* rate and has no terminal top-rate row. Hand-traced example: gross тВм100,000 currently computes **тВм11,188** income tax; the correct figure (even with only the stale thresholds) is **тВм25,229** тАФ a ~55% under-count. This is a logic bug, not a stale-number issue. Fix shape: `[[0,0],[11294,.11],[28797,.30],[82341,.41],[177106,.45]]` (also adds the missing 45% top band).
2. **US federal tax stops at a 32% top bracket, applied forever above $191,950** тАФ the 35% and 37% brackets don't exist in the code at all, understating tax for any high annualized gross (relevant since the app can annualize at the user's full working ceiling, not just typical hours).
3. **Canada federal tax stops at 26%** тАФ the 29% and 33% top brackets are entirely missing, same understatement pattern. CPP2 (a second contribution tier above the original CPP ceiling) is unmodeled entirely, not just outdated.
4. **Australia stops at 37%** тАФ the 45% top bracket above $190,000 is missing. **Also, independently discovered while implementing (not caught by the original research pass, which compared threshold numbers but didn't hand-trace loop execution the way it did for France): Australia's bracket array has the exact same shifted-row bug as France.** The shipped array `[[18200,0],[45000,.16],[135000,.30],[190000,.37]]` pairs the $18,200 threshold with the 0% rate meant for the band *below* it, pushing every real rate one band late and leaving the top band uncovered. Hand-traced: gross $100,000 computes **$8,800**; correct tax (even under the old 2024-25 rates) is **$20,788** тАФ more than 50% under-counted. Fix requires an explicit leading `[0,0]` entry, same pattern as the France fix: `[[0,0],[18200,.15],[45000,.30],[135000,.37],[190000,.45]]`.
5. **Germany's flat 30% "approximation" materially misrepresents the real curve**, not just a rounding simplification: it overtaxes low/mid incomes (real effective rate ~8-18% around тВм20k-40k) and only roughly matches near тВм70k-100k, while understating tax for higher earners subject to 42-45%. A flat rate flattens away the progression the app's own "effective-rate estimate" disclaimer implies still exists.
6. **Netherlands omits arbeidskorting (labour tax credit) entirely** тАФ for an employee this credit is larger than the general credit already modeled (2026 max ~тВм5,685), so its absence materially overstates Dutch tax specifically for the employee/shift-worker case this app is built for.

## Stale figures (structure is fine, numbers need updating)

**UK** тАФ Personal Allowance (┬г12,570) and higher-rate threshold (┬г50,270) are correct and frozen through 2026/27; NI 8%/2% bands match current rates exactly. Only gap: no additional-rate (45% above ┬г125,140) band тАФ add a third band, the personal-allowance taper above ┬г100k is fine to keep skipped.

**Ireland** тАФ standard-rate cutoff should be тВм44,000 (code has тВм42,000); USC second band should end at тВм28,700 not тВм25,760, at 2% not... actually the rate above that band should be 3% not 4% (code currently applies 4% where 2026 law says 3%); PRSI should be ~4.2% (code has 4.1%, stale but close).

**Germany** тАФ Grundfreibetrag should be тВм12,348 (code has тВм11,604, a 2024 figure); top marginal rate (42%) now starts at тВм69,878. Social contribution rate should be ~21% not 20%; the single earnings cap is outdated and real law actually splits into two caps (тВм69,750 health/care, тВм101,400 pension/unemployment) тАФ fine to keep one blended cap if updated, e.g. ~тВм85k.

**US** тАФ standard deduction should be $16,100 (code has $14,600, a 2024 figure); brackets should be 10% to $12,400, 12% to $50,400, 22% to $105,700, 24% to $201,775, 32% to $256,225 (then 35%/37%, see bug #2 above). FICA should cap the 6.2% Social Security portion at the $184,500 wage base (Medicare's 1.45% stays uncapped) тАФ code currently applies the full 7.65% uncapped.

**Canada** тАФ basic personal amount should be $14,829 (code has $15,705, stale); lowest bracket rate is now 14% not 15% (cut in 2025, permanent for 2026); bracket thresholds should be $58,523 / $117,045 / $181,440 (then 29%/33%, see bug #3). CPP should be 5.95% up to $74,600 (not 68,500), EI 1.63% up to $68,900 (a different, separate cap from CPP's) тАФ not a flat combined 7.3%.

**Australia** тАФ tax-free threshold ($18,200) still correct. Second bracket should be 15% not 16% (cut effective 1 July 2026); bracket thresholds should be $45,000 / $135,000 / $190,000 (then 45%, see bug #4). Medicare levy cliff threshold should be $29,207 not $27,222 (real law has a shade-in band to $36,509 before the flat 2%; keeping a single cliff is a fine simplification once the number's current).

**France** тАФ thresholds should be 0% to тВм11,600, 11% to тВм29,579, 30% to тВм84,577, 41% to тВм181,917 (then 45%, see bug #1) тАФ current code is using 2024-income (bar├иme 2024) figures, not 2026. Cotisations sociales ~22% flat remains a fair round figure, no change needed.

**Netherlands** тАФ box 1 should be 3 bands not 2: 35.75% to тВм38,883, 37.56% to тВм78,426, 49.50% above (2026). General tax credit should be max тВм3,115, phasing from тВм29,736 at 6.398% (code's тВм3,362/тВм24,000/6% figures are stale). Bundling income tax + national insurance into one box-1 rate is real and correct, not a simplification error тАФ that's how Dutch payroll actually works.

**Spain** тАФ IRPF brackets (19/24/30/37/45 at 0/12,450/20,200/35,200/60,000) are current and correct, including the choice to ignore regional variation тАФ no change needed there. Missing the 47% bracket above тВм300,000 (minor). Social security ceiling should be тВм61,214.40 for 2026 (code has тВм56,646, a 2024 figure); rate should be ~6.47-6.50% including the MEI surcharge (code has 6.35%, predates MEI).
